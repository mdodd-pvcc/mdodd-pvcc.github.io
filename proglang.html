Programming Terms and Definitions

Term Definition
Machine Language The lowest-level programming language consisting of binary code that is directly executed by the computer's CPU.
Debugging The process of finding and fixing errors or bugs in a program.
Assembler A program that translates assembly language code into machine language code.
Assembly Language A low-level programming language that uses mnemonics to represent machine instructions, making it easier for humans to read and write.
Translator A program that translates code from one programming language into another.
Compiler A program that translates high-level programming language code into machine language code, typically producing an object program.
Interpreter A program that reads and executes code line-by-line, translating each line into machine language code on the fly.
Cross-platform software Software that can run on multiple operating systems or platforms.
Natural Language A language that is spoken or written by humans, such as English or Spanish.
Formal Language A language that is designed for specific purposes, such as programming languages or mathematical notation.
Machine Independence The ability of a program to run on different computer systems without modification.
High-Level Programming Language A programming language that is closer to human language, making it easier to read and write than low-level programming languages.
Object-Oriented Programming (OOP) A programming paradigm that focuses on creating objects that have properties and methods, allowing for code reusability and modularity.
Scripting Language A programming language that is used for scripting or automating tasks, often interpreted rather than compiled.
Data Types The types of data that can be stored and manipulated in a program.
Integer A data type that represents whole numbers, such as 1, 2, 3.
Float A data type that represents decimal numbers, such as 1.5
String A data type that represents a sequence of characters, such as "hello world".
Primitive Data Type A basic data type that is not composed of other data types, such as integers, floats, and strings.
Data Structure A way of organizing and storing data in a computer program, such as arrays, linked lists, and trees.
Python Collection Data Structures A set of built-in data structures in Python that can hold multiple values, such as lists, tuples, sets, and dictionaries.
List A collection of values that are ordered and changeable, represented by square brackets [].
Tuple A collection of values that are ordered and unchangeable, represented by parentheses ().
Set A collection of unique values that are unordered, represented by curly braces {}.
Dictionary A collection of key-value pairs that are unordered, represented by curly braces {} with colons separating the keys and values.
Function A block of code that performs a specific task and can be called multiple times throughout a program.
Local Variable A variable that is declared and used within a specific function or block of code, and is not accessible outside of that function or block.
Global Variable A variable that is declared outside of a function or block of code, and can be accessed and modified from anywhere within a program.
Source Program The original code that a programmer writes, often in a high-level programming language.
Object Program The compiled or assembled version of a source program that can be executed by a computer.
