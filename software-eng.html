Chapter 7: Software Engineering

Term Definition
Software engineering The process of designing, creating, testing, and maintaining software using a systematic approach that incorporates principles of computer science, engineering, and project management.
Integrated Development Environment (IDE) A software application that provides comprehensive facilities to computer programmers for software development.
Association for Computing Machinery (ACM) A professional organization dedicated to advancing computing as a science and profession.
  • Purpose of ACM: To promote the development and advancement of computing and information technology.
  • Benefits of ACM membership: Access to conferences, workshops, and publications; networking opportunities; and career development resources.
  • Student ACM membership: Reduced membership rates and access to student-specific resources.
Institute of Electrical and Electronics Engineers (IEEE) A professional organization dedicated to advancing technology for the benefit of humanity.
  • Purpose of IEEE: To promote the development and advancement of technology in electrical and electronics engineering and related fields.
  • Benefits of IEEE membership: Access to conferences, workshops, and publications; networking opportunities; and career development resources.
  • Student IEEE membership: Reduced membership rates and access to student-specific resources.
Software Life Cycle: Development phase The phase in software development where the software is actually developed.
  • Requirements analysis: The process of gathering and documenting requirements for the software.
  • Design: The process of creating a plan for the software, including its architecture and user interface.
  • Implementation: The process of writing the code for the software.
  • Testing: The process of ensuring that the software works as intended and is free of bugs.
Modular implementation The practice of breaking down software into smaller, independent modules that can be developed and tested separately.
Unified Modeling Language (UML) A standardized modeling language for software development.
  • Use case: A description of how a user will interact with the software.
  • Use case diagram: A visual representation of the use cases for the software.
Data relationships The way in which data elements are related to each other in a database or other data storage system.
  • One-to-one: A relationship between two data elements where each element is associated with only one instance of the other element.
  • One-to-many: A relationship between two data elements where one element is associated with multiple instances of the other element.
  • Many-to-many: A relationship between two data elements where each element is associated with multiple instances of the other element.
Design pattern A general reusable solution to a commonly occurring problem in software design.
Software quality assurance (SQA) The process of ensuring that software meets specified quality standards before it is released to customers.
Software testing The process of evaluating software to ensure that it performs as intended and is free of bugs and other defects.
  • Glass box testing: A testing technique that involves examining the internal structure of the software to ensure that it functions correctly.
  • Black box testing: A testing technique that involves evaluating the software's external behavior without examining its internal structure.
  • Beta testing: A testing technique where the software is released to a limited audience for real-world testing and feedback.
  • Alpha testing: A testing technique where the software is tested in-house by a select group of users before it is released to the public.
Documentation The process of creating and maintaining documentation for software.
  • User documentation: Documentation that is written for end users of the software to help them understand how to use it.
  • System documentation: Documentation that describes the technical details of the software, including its architecture and design.
  • Technical documentation: Documentation that is written for developers and other technical users to help them understand how to work with the software.
Intellectual property law The body of law that governs the creation, ownership, and use of intellectual property, including patents, trademarks, and copyrights.
  • Software license: A legal agreement that governs the use of software, typically defining how the software can be used and distributed, and any restrictions or limitations on those uses.
  • Software disclaimer: A statement that disclaims any warranties or guarantees regarding the software, and limits the liability of the software creator or distributor in case of problems or damages arising from the use of the software.
  • Best defenses against software liability claims: Steps that software creators and distributors can take to minimize their exposure to liability claims, such as obtaining adequate insurance coverage, including appropriate disclaimers in the software license, and regularly updating and testing the software to minimize the risk of bugs or other problems.
